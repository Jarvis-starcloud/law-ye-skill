# 电子证据固定方法

> 定位：各类电子证据的固定操作SOP，确保法院采纳
> 创建时间：2026-04-29
> 版本：v1.0 | Phase 2 模块3交付物
> 法律依据：《最高人民法院关于民事诉讼证据的若干规定》（2019修正）+《电子数据规定》

---

## 一、电子证据的法律地位

### 1.1 法律定义

《民事诉讼法》第66条将**电子数据**列为独立证据类型。

《最高人民法院关于民事诉讼证据的若干规定》第14条明确电子数据包括：
- 网页、博客、微博等网络平台发布的信息
- 手机短信、电子邮件、即时通信（微信/QQ等）
- 用户注册信息、身份认证信息、电子交易记录
- 文档、图片、音频、视频、数字证书、计算机程序等

### 1.2 法院审查标准

法院从以下五个方面审查电子证据：

| 审查要素 | 内容 |
|----------|------|
| **生成、存储、传输的可靠性** | 系统是否正常运行 |
| **保存的完整性** | 是否被篡改 |
| **取得方式的合法性** | 是否侵犯他人权利 |
| **与案件关联性** | 与待证事实的关系 |
| **提供者的可信度** | 提供者是否有利害关系 |

## 二、微信聊天记录固定（最常见）

### 2.1 完整固定流程

```
Step 1: 身份验证（证明"对面是谁"）
├── 打开微信 → 点击对方头像 → 进入个人信息页
├── 截图显示：微信号、手机号、备注名
├── 如果是微信群：截图群成员列表
└── 全程录屏

Step 2: 聊天内容展示（证明"说了什么"）
├── 从头开始翻看聊天记录
├── 手指滑动展示，不要跳跃
├── 关键内容停顿3秒以上
├── 涉及语音消息：播放并录制
└── 全程录屏

Step 3: 静态截图保存
├── 逐屏截图所有相关内容
├── 确保每条消息的时间戳可见
├── 语音消息截图+文字转写
└── 图片/视频消息单独保存

Step 4: 文字整理
├── 按时间顺序整理为文本
├── 格式：[时间] [发言人]: [内容]
├── 标注关键信息："此处对方确认欠款金额"
└── 打印备用

Step 5: 备份与公证（重要案件）
├── 原始录屏文件备份至电脑+云盘
├── 截图文件多处备份
└── 标的额大或证据可能灭失→做公证保全
```

### 2.2 微信转账记录固定

| 步骤 | 操作 |
|------|------|
| 1 | 微信→我→服务→钱包→账单 |
| 2 | 找到目标交易→点击进入详情 |
| 3 | 截图详情页（含交易单号、金额、时间、对方信息） |
| 4 | 导出交易账单（微信支持导出3个月内的账单） |
| 5 | 全程录屏 |

### 2.3 常见问题

| 问题 | 答案 |
|------|------|
| 对方删了聊天记录怎么办？ | 你手机上的记录仍有效；如需恢复，可申请法院调取腾讯服务器数据 |
| 对方改了微信名怎么办？ | 提前截图验证身份；微信号不可更改，以微信号为准 |
| 语音消息法院认吗？ | 认，但需在原始设备上播放展示 |
| 群聊记录有效吗？ | 有效，但需证明群内发言人的真实身份 |

## 三、电子邮件固定

### 3.1 标准操作

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1 | 登录邮箱（浏览器） | 展示邮箱账号所有权 |
| 2 | 打开目标邮件 | 展示完整邮件内容 |
| 3 | 查看邮件头信息 | 展示发件人/IP/时间等元数据 |
| 4 | 全文截图 | 含发件人、收件人、主题、时间、正文 |
| 5 | 导出eml文件 | 保留原始邮件数据（含邮件头） |
| 6 | 打印邮件 | 提交法院的纸质版 |

### 3.2 邮件头信息的价值

邮件头包含：
- **From**：发件人地址
- **To**：收件人地址
- **Date**：发送时间（服务器时间）
- **Message-ID**：唯一标识
- **Received**：传输路径（可追溯IP地址）

> **实务提醒**：邮件头信息是证明邮件真实性的核心。对方如果说"我没发过这封邮件"，邮件头中的传输路径可以证明来源。

## 四、录音证据固定

### 4.1 合法录音的条件

| 条件 | 说明 |
|------|------|
| ✅ 你是对话参与者 | 自己参与的对话可以录音 |
| ✅ 在公共场合录音 | 公开谈话没有隐私期待 |
| ❌ 偷录他人对话 | 你不在场偷录第三方对话=不合法 |
| ❌ 窃听设备 | 安装窃听器、木马程序=违法 |
| ❌ 非法侵入 | 非法进入他人住宅/办公室录音=违法 |

### 4.2 录音操作规范

**录音前**：
- 测试录音设备（手机自带录音即可）
- 确认存储空间充足
- 记录时间、地点

**录音中的关键技巧**：

| 技巧 | 示例 |
|------|------|
| **开头自报日期地点** | 自言自语"今天4月29号，在公司会议室" |
| **让对方自报身份** | "张总，你说的这个事..." |
| **确认关键事实** | "你的意思是公司决定解除我的劳动合同？" |
| **确认关键金额** | "所以你说欠的是三个月工资一共三万块？" |
| **不诱导不威胁** | 正常对话即可，不要故意设套 |

**录音后**：
- 原始文件不做任何处理
- 备份至3个以上位置（手机+电脑+云盘）
- 整理完整文字稿
- 标注时间戳和关键节点

### 4.3 录音证据的提交格式

```
证据X：与[对方姓名]的通话录音

录音时间：2026年X月X日 XX:XX
录音地点：[具体地点]
录音时长：XX分XX秒
录音设备：[手机型号]
录音人：[你的姓名]

文字整理（关键部分）：

[00:15] 我（申请人）：张总，关于上个月工资的事...
[00:22] 张XX（被申请人法定代表人）：工资的事你别急...
[01:05] 张XX：公司最近确实资金紧张，三月的工资确实还没发...
        ↑ 关键证据点：被申请人承认拖欠3月工资
[02:30] 我：那大概什么时候能发？
[02:35] 张XX：我也不确定，五月吧...

（完整文字稿见附件）
```

## 五、网页/APP内容固定

### 5.1 网页内容

| 方法 | 操作 | 证明力 |
|------|------|--------|
| 截图 | 全页截图（含URL栏和时间）| ⭐⭐ |
| 打印网页 | 浏览器打印为PDF（含URL和打印日期）| ⭐⭐⭐ |
| 网页存档 | 使用 archive.org 或类似服务存档 | ⭐⭐⭐ |
| **公证保全** | 公证员操作电脑访问网页并出具公证书 | ⭐⭐⭐⭐⭐ |

### 5.2 APP内容（电商/外卖/打车等）

| 步骤 | 操作 |
|------|------|
| 1 | 打开APP展示登录账号 |
| 2 | 进入目标订单/内容页面 |

| 4 | 逐屏截图关键信息 |
| 5 | 导出订单记录（如APP支持）|
| 6 | 必要时做公证 |

## 六、区块链存证（新兴方式）

### 6.1 什么是区块链存证

将电子证据的哈希值上链，利用区块链不可篡改的特性证明证据在某个时间点已经存在且未被修改。

### 6.2 法院认可情况

- **最高人民法院已明确认可**区块链存证的法律效力（2018年《关于互联网法院审理案件若干问题的规定》第11条）
- 北京互联网法院"天平链"、杭州互联网法院"司法区块链"已大量应用

### 6.3 常用平台

| 平台 | 运营方 | 特点 |
|------|--------|------|
| 至信链 | 腾讯+北京互联网法院 | 微信生态对接好 |
| 天平链 | 北京互联网法院 | 司法机关直接运营 |
| 蚂蚁链 | 蚂蚁集团 | 支付宝/淘宝证据 |

### 6.4 操作流程（以至信链为例）

1. 注册至信链账号并实名认证
2. 上传需要存证的文件（截图/录屏/文档等）
3. 系统生成文件哈希值并上链
4. 获得**存证编号**和**区块链存证证书**
5. 诉讼时向法院提交存证编号，法院可在线核验

## 七、证据固定时机与优先级

### 7.1 取证黄金时间表

| 优先级 | 时机 | 要做什么 |
|--------|------|----------|
| 🔴 立即 | 发现侵权/收到通知的**当天** | 截图+录屏+备份核心证据 |
| 🟡 24h内 | 争议发生后**一天内** | 整理证据清单+补充录音/截图 |
| 🟢 一周内 | 决定维权后 | 公证保全+银行流水+企业信息打印 |
| ⚪ 起诉前 | 提交材料前 | 编制证据目录+文字整理+复印装订 |

### 7.2 容易灭失的证据（优先固定）

| 证据类型 | 灭失风险 | 固定方法 |
|----------|----------|----------|
| 微信聊天记录 | 对方可能删除/清空 | 立刻录屏+截图 |
| 网页内容 | 对方可能下架/修改 | 截图+网页存档+公证 |
| 公司内部系统 | 离职后无法访问 | 在职时导出/截图 |
| 监控录像 | 通常只保存15-30天 | 尽快申请调取或复制 |
| 证人记忆 | 时间越久越模糊 | 尽早取得书面证言 |
